Unit Segmentation for EMU Condition MonitoringThis directory contains the visual model code for Electric Multiple Units (EMU) component segmentation and image-text pair generation. This module implements unsupervised unit segmentation via Masked Image Modeling (MIM) based on the BEiT-v2 framework , combined with a physics-informed knowledge base filter to generate descriptive prompts for downstream Multimodal Large Language Model (MLLM) fine-tuning.  🛠️ Installation GuideThe core vision model of this project is developed based on the official BEiT-v2 repository.Base Environment Setup: Please refer to the official BEiT-v2 repository to set up the foundational environment.Replace Custom Files: Overwrite the files in your cloned official repository with the customized files provided in this project.⚠️ Important Environment Setup Warning
If you plan to use Open-CLIP or DINO as the teacher model for extracting semantic features, it is highly recommended to create a new, separate Conda virtual environment. During our experiments, we encountered an irreconcilable version conflict between open-clip and the BEiT-v2 evaluation codebase. Sharing the same environment will result in runtime errors.  📂 Core Directory Overview: semantic_segmentationThis directory is primarily used for post-processing and data preparation. It contains two key functional modules:Knowledge-Based Filter: Contains a simplified, physics-rule-based post-processing script. Due to intellectual property and industrial confidentiality requirements, specific mechanical structure designs and absolute numerical dimensions (e.g., specific scaling coefficients for the dynamic sliding window) have been omitted. This code demonstrates how to use topological connectivity and morphological spatial constraints to filter out incorrect predictions generated by the visual model.  Prompt Generation: Utilizes high-quality segmentation results combined with sensor data to automatically generate rich textual descriptions containing structural information and anomaly context.  ⚙️ Hardware Configuration & Hyperparameter AdjustmentDefault Hardware Configuration: The default hyperparameters in the current code (such as batch_size=64 or 128) are configured for a single node with 8 GPUs (our experiments were conducted on a cluster of eight NVIDIA A10 GPUs).  Scaling Adjustments: If the number of GPUs or VRAM capacity on your server differs from ours and you need to adjust the batch_size, please ensure you strictly adjust the learning rate proportionally to guarantee proper model convergence.🚀 Example Execution CommandsBelow are the command examples for the two-stage pre-training (visual tokenizer and MIM pre-training) and prompt generation:1. Stage 1: Visual Tokenizer (VQ-KD) TrainingThis stage trains a tokenizer to encode EMU part images into discrete semantic codes.  Bashpython -m torch.distributed.launch --nproc_per_node=8 run_vqkd_training.py \
    --data_path /path/to/your/emu/dataset \
    --output_dir /path/to/save/tokenizer \
    --teacher_model clip \
    --batch_size 64 \
    --lr 5e-5 \
    --epochs 100 \
    --codebook_size 8192
2. Stage 2: Masked Image Modeling (MIM) Pre-trainingIn this stage, the vision model learns how individual parts assemble into complete units in an unsupervised manner by predicting the discrete codes of masked image patches.  Bashpython -m torch.distributed.launch --nproc_per_node=8 run_mim_pretraining.py \
    --data_path /path/to/your/emu/dataset \
    --output_dir /path/to/save/mim_model \
    --tokenizer_weight /path/to/save/tokenizer/checkpoint.pth \
    --batch_size 128 \
    --lr 7.5e-4 \
    --epochs 300
3. Prompt GenerationAfter completing the segmentation inference, navigate to the semantic_segmentation directory to generate image-text pairs for MLLM fine-tuning by combining knowledge rules and clustering results.Bashcd semantic_segmentation

python generate_prompts.py \
    --segmentation_results /path/to/segmentation/outputs.json \
    --sensor_metadata /path/to/sensor/data.csv \
    --output_file /path/to/save/mllm_training_pairs.json \
